# Цифровой Совет Директоров

**Платформа электронного управления заседаниями совета директоров**

---

## Описание проекта

«Цифровой Совет Директоров» — это изолированное Enterprise-решение для организации и проведения заседаний совета директоров акционерных обществ и обществ с ограниченной ответственностью. Платформа обеспечивает легитимное электронное голосование, документооборот и управление комитетами в полном соответствии с корпоративным законодательством Российской Федерации.

---

## Проблема, которую решает проект

Традиционное управление советом директоров связано с множеством проблем:

- Бумажный документооборот требует физического присутствия или почтовых пересылок
- Подсчёт голосов осуществляется вручную с высоким риском ошибок
- Сложно обеспечить легитимность заочных голосований
- Отсутствует единое рабочее пространство для директоров
- Трудно обеспечить соответствие требованиям 208-ФЗ и 14-ФЗ
- Решения принимаются дольше из-за логистических задержек

## Ключевые преимущества

- **Юридическая значимость**: Электронные подписи (ПЭП/УКЭП) равнозначны собственноручным
- **Соответствие 208-ФЗ**: Автоматический контроль кворума и фиксация участников
- **Офлайн-работа**: Голосование без интернета с последующей синхронизацией
- **ГОСТ-документы**: Автоматическая генерация протоколов по ГОСТ Р 7.0.97-2025
- **Безопасность**: Некорректируемый аудит-лог и защита данных по 152-ФЗ
- **Гибкость**: Динамическое управление комитетами без изменения кода

## Основные сценарии использования

### 1. Организация заседания
1. Корпоративный секретарь создаёт уведомление о созыве
2. Автоматически формируется повестка с вопросами
3. Директора получают уведомления по email
4. Система автоматически считает кворум

### 2. Электронное голосование
1. Директора голосуют «За», «Против» или «Воздержался»
2. Голос фиксируется электронной подписью (ПЭП/УКЭП)
3. Система автоматически считает результаты
4. При отсутствии кворума — автоматическая блокировка

### 3. Офлайн-голосование
1. Материалы заседания скачиваются в зашифрованный кэш
2. Директор голосует в офлайн-режиме
3. При появлении сети — автоматическая синхронизация
4. Все действия фиксируются с UTC-метками

### 4. Работа комитетов
1. Создание комитетов через админ-панель
2. Назначение председателей и членов
3. Формирование заключений с агрегацией мнений
4. Подписание протоколов комитета

### 5. Генерация документов
1. Автоматическое формирование протокола по ГОСТу
2. Наложение водяных знаков на документы
3. Подписание УКЭП председателем
4. Архивирование в журнале аудита
