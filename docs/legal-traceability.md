# Матрица трассируемости требований законодательства (Бизнес-уровень)

Этот документ фиксирует **бизнес/юридический уровень** архитектуры —
явное соответствие норм федеральных законов конкретным модулям, сервисам
и статусу их реализации. Ведение матрицы закреплено решением **ADR-008**
в [`docs/architecture.md`](architecture.md#adr-008-явное-отображение-бизнесюридического-уровня-в-архитектуре).

Назначение документа:

- дать однозначный ответ на вопрос "где в архитектуре видно, что закон X
  соблюдён";
- служить чек-листом при аудите ИБ и юридической проверке;
- фиксировать разрыв (gap) между требованием закона и фактическим статусом
  реализации, чтобы он был виден архитектору и product-owner'у, а не
  терялся в коде.

---

## Легенда статусов

| Статус | Значение |
|--------|----------|
| ✅ Реализовано | Требование закона полностью покрыто кодом/процессом |
| 🟡 Частично | Есть базовая реализация, но не покрывает требование целиком |
| 🟠 Организационно | Требование закрывается не кодом, а организационной мерой (хостинг, регламент, договор) |
| ❌ Не реализовано | Требование не покрыто, зафиксировано в "Известные проблемы" (`AGENTS.md`) |

## Порядок актуализации матрицы

1. При изменении требований законодательства (новая редакция 208-ФЗ,
   14-ФЗ, 63-ФЗ, 152-ФЗ) — обновляется соответствующий раздел этого файла
   **до** внесения изменений в код.
2. При добавлении нового модуля/сервиса, реализующего юридическое
   требование, — обязательно добавляется строка в соответствующую таблицу
   с указанием статуса.
3. Изменение статуса с ❌/🟡 на ✅ допускается только после подтверждения
   юридическим отделом (см. "Известные проблемы" в `AGENTS.md` —
   ГОСТ-шаблоны и интеграция с КриптоПро требуют такого утверждения).
4. Ответственный за актуальность матрицы — корпоративный секретарь
   (бизнес-часть) совместно с архитектором проекта (техническая часть).
